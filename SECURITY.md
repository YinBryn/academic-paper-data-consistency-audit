# Security Policy

## Supported versions

This project is currently in alpha. Security and responsible-use reports should target the current `main` branch unless a release tag is explicitly affected.

## Reporting a vulnerability

If you find a security issue in the code, packaging, documentation, or example workflows, please open a private report if available through GitHub Security Advisories. If private reporting is not available, open a minimal public issue that avoids exploit details and clearly marks the report as security-related.

## Data and privacy concerns

This project should not contain:

- copyrighted publisher PDFs
- proprietary datasets
- private emails or personal contact information
- real paper data in synthetic examples
- author-intent allegations or misconduct labels

If such material appears in the repository, please report it so it can be removed or rewritten.

## Responsible disclosure expectations

Please provide:

- affected file or workflow
- minimal steps to reproduce
- potential impact
- suggested mitigation, if known

The maintainers will aim to address valid reports promptly and transparently.
