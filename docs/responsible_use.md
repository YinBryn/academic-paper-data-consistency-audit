# Responsible Use

This project is intended for technical clarification and reproducibility-oriented review.

## Core principles

- Keep comments technical and evidence-based.
- Distinguish observation, recalculation, and interpretation.
- Do not infer author intent.
- Do not use accusatory language.
- Do not redistribute copyrighted PDFs or proprietary datasets.
- Give authors and journals room to clarify.
- Use source-data links rather than re-uploading publisher files whenever possible.

## Preferred wording

Use wording such as:

- `requires clarification`
- `appears to differ`
- `recalculated value`
- `reported value`
- `same sample and condition`
- `evidence-claim alignment`

Avoid wording that asserts intent or misconduct.

## Public comments

For public post-publication review, use one issue per comment:

1. exact location
2. factual observation
3. reproducible check
4. technical significance
5. clarification request

## Data handling

Use synthetic examples for documentation. For real audits, avoid uploading publisher PDFs or proprietary source files into public repositories.
